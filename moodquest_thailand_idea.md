# MoodQuest Thailand

## แนวคิดหลัก

**MoodQuest Thailand** คือแพลตฟอร์มท่องเที่ยวที่ไม่ได้เริ่มจากคำถามว่า “อยากไปที่ไหน” แต่เริ่มจากคำถามว่า **“วันนี้รู้สึกอยากเที่ยวแบบไหน”**

ระบบจะใช้ AI ช่วยสร้างเส้นทางท่องเที่ยวตามอารมณ์ ความสนใจ เวลา งบประมาณ และพื้นที่ที่ผู้ใช้เลือก จากนั้นเปลี่ยนทริปธรรมดาให้กลายเป็น **ภารกิจ / เกม / ประสบการณ์ส่วนตัว**

---

## ปัญหาที่พบในแพลตฟอร์มท่องเที่ยวทั่วไป

แพลตฟอร์มท่องเที่ยวส่วนใหญ่ในปัจจุบันมักเน้นเรื่อง:

- ค้นหาสถานที่ท่องเที่ยว
- จองโรงแรม
- จองตั๋ว
- จองกิจกรรม
- อ่านรีวิว
- ดูแผนที่

แต่ยังมีช่องว่างคือ ผู้ใช้หลายคนไม่ได้ต้องการแค่ “ข้อมูลเยอะ” แต่ต้องการ **ประสบการณ์ที่เหมาะกับอารมณ์และสถานการณ์ของตัวเอง**

ตัวอย่างเช่น:

- วันนี้เหนื่อย อยากเที่ยวแบบฮีลใจ
- อยากไปคนเดียวแบบไม่อึดอัด
- อยากถ่ายรูปสวย ๆ แต่ไม่อยากไปที่แมส
- อยากพาพ่อแม่เที่ยวแบบไม่เหนื่อย
- อยากเที่ยวครึ่งวันแบบไม่ต้องคิดเยอะ
- อยากมีเรื่องเล่ากลับมา ไม่ใช่แค่ไปถ่ายรูป

---

## จุดขายของระบบ

MoodQuest Thailand ไม่ได้เป็นแค่ Trip Planner แต่เป็นระบบที่ช่วยเปลี่ยนการเที่ยวให้มีความหมายมากขึ้น

### จุดต่างจากแพลตฟอร์มทั่วไป

| แพลตฟอร์มทั่วไป | MoodQuest Thailand |
|---|---|
| เริ่มจากสถานที่ | เริ่มจากอารมณ์ของผู้ใช้ |
| แนะนำว่าไปไหนดี | แนะนำว่าควรไป ทำอะไร และทำไม |
| เน้นข้อมูลและการจอง | เน้นประสบการณ์ส่วนตัว |
| ทริปเหมือนกันได้ง่าย | ทริปเปลี่ยนตาม mood และ persona |
| จบทริปแล้วไม่มีอะไรต่อ | มี badge, memory, mission และ story |

## ตัวอย่างการใช้งาน

### Input จากผู้ใช้

```text
Mood: เหนื่อย อยากฮีลใจ
พื้นที่: กรุงเทพ
เวลา: 4 ชั่วโมง
งบประมาณ: 800 บาท
การเดินทาง: BTS / MRT / เดิน
```

### Output จากระบบ

```text
ชื่อทริป: Slow Bangkok
ระยะเวลา: 4 ชั่วโมง
งบประมาณโดยประมาณ: 650 บาท
ระดับความเหนื่อย: ต่ำ
เหมาะกับ: คนที่อยากพักใจแบบไม่ต้องออกนอกเมือง
```

### Mission ที่ระบบสร้างให้

```text
Mission 1: เดินเล่นริมแม่น้ำและถ่ายรูปวิวที่ทำให้รู้สึกสงบ
Mission 2: ลองขนมไทย 1 อย่าง แล้วให้ AI เล่าเรื่องที่มาของเมนูนั้น
Mission 3: หามุมเงียบ ๆ นั่งพัก 10 นาที โดยไม่เปิดโซเชียล
Mission 4: ถ่ายภาพพระอาทิตย์ตกหรือแสงเย็นของเมือง
```

### Badge ที่ได้รับ

```text
Badge: Urban Healer
คำอธิบาย: คุณปลดล็อกการเที่ยวแบบฮีลใจในเมืองใหญ่
```

## ฟีเจอร์หลัก

## 1. Mood-Based Trip Generator

ระบบให้ผู้ใช้เลือกอารมณ์หรือสไตล์การเที่ยว แล้ว AI สร้างทริปให้เหมาะกับอารมณ์นั้น

ตัวอย่าง:

- ฮีลใจ → สถานที่เงียบ เดินทางง่าย คนไม่เยอะ
- สายกิน → ร้านท้องถิ่น เมนูเฉพาะพื้นที่ ตลาดเก่า
- ถ่ายรูป → จุดวิวดี แสงสวย คาเฟ่ดีไซน์เด่น
- ครอบครัว → เดินน้อย มีที่นั่งพัก ห้องน้ำสะดวก
- งบน้อย → สถานที่ฟรี เดินทางด้วยขนส่งสาธารณะ

---

## 2. AI Mission Generator

ระบบไม่ได้แค่บอกว่าให้ไปสถานที่ไหน แต่สร้าง “ภารกิจ” ให้ทำในแต่ละจุด

ตัวอย่างภารกิจ:

- ถ่ายรูปสิ่งที่สะท้อนความเป็นท้องถิ่น
- ลองเมนูที่คนพื้นที่นิยม
- หาป้ายเก่า อาคารเก่า หรือรายละเอียดที่คนทั่วไปมองข้าม
- ตอบคำถามสั้น ๆ เกี่ยวกับสถานที่
- เช็กอินเมื่อถึงจุดหมาย

---

## 3. AI Storytelling

เมื่อผู้ใช้ไปถึงสถานที่ ระบบจะเล่าเรื่องแบบสั้น เข้าใจง่าย และมีอารมณ์ร่วม

ตัวอย่าง:

```text
คุณกำลังอยู่ในย่านเก่าที่เคยเป็นจุดค้าขายสำคัญของเมือง
ภารกิจของคุณคือมองหาร่องรอย 3 อย่างที่บอกว่าย่านนี้เคยรุ่งเรือง
```

---

## 4. Photo Check-in / AI Verification

ผู้ใช้สามารถถ่ายรูปเพื่อยืนยันภารกิจ ระบบใช้ AI ช่วยตรวจเบื้องต้นว่ารูปตรงกับภารกิจหรือไม่

ตัวอย่าง:

- ถ่ายรูปหน้าสถานที่
- ถ่ายรูปอาหารท้องถิ่น
- ถ่ายรูปวิว
- ถ่ายรูปวัตถุหรือจุดสังเกตตามภารกิจ

ระยะ MVP อาจยังไม่ต้องทำ AI ตรวจละเอียด ใช้การอัปโหลดรูป + กด complete ก่อนก็ได้

---

## 5. Badge & Achievement

เมื่อทำภารกิจสำเร็จ ผู้ใช้จะได้รับ badge

ตัวอย่าง badge:

- Urban Healer
- Local Food Hunter
- Hidden Photo Spot Finder
- Slow Traveler
- Culture Explorer
- Budget Trip Master
- Rainy Day Survivor

---

## 6. Travel Memory Summary

หลังจบทริป AI สรุปเป็นบันทึกการเดินทางสั้น ๆ ให้ผู้ใช้แชร์ได้

ตัวอย่าง:

```text
วันนี้คุณเลือกเที่ยวแบบฮีลใจในกรุงเทพฯ คุณเดินทางผ่านย่านริมแม่น้ำ ลองขนมไทย และใช้เวลาสั้น ๆ เพื่อพักจากความวุ่นวายของเมือง ทริปนี้คุณได้รับ Badge: Urban Healer
```

## Database เบื้องต้น

### ตาราง places

```sql
CREATE TABLE places (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL,
    province TEXT NOT NULL,
    category TEXT,
    mood_tags TEXT[],
    lat FLOAT,
    lng FLOAT,
    avg_stay_minutes INT,
    estimated_cost INT,
    description TEXT
);
```

### ตาราง missions

```sql
CREATE TABLE missions (
    id SERIAL PRIMARY KEY,
    place_id INT,
    mood TEXT,
    mission_title TEXT,
    mission_description TEXT,
    difficulty TEXT,
    badge_reward TEXT
);
```

### ตาราง user_trips

```sql
CREATE TABLE user_trips (
    id SERIAL PRIMARY KEY,
    trip_title TEXT,
    mood TEXT,
    province TEXT,
    budget INT,
    duration_hours INT,
    transport_mode TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### ตาราง trip_items

```sql
CREATE TABLE trip_items (
    id SERIAL PRIMARY KEY,
    trip_id INT,
    place_id INT,
    order_no INT,
    mission_text TEXT,
    is_completed BOOLEAN DEFAULT FALSE
);
```

## สรุป

MoodQuest Thailand คือแนวคิดแพลตฟอร์มท่องเที่ยวที่ต่างจาก Trip Planner ทั่วไป เพราะไม่ได้เน้นแค่ “จัดแผนเที่ยว” แต่เน้นการสร้าง **ประสบการณ์ส่วนตัวแบบมีภารกิจ มีเรื่องเล่า และมีความรู้สึก**

จุดแข็งของไอเดียนี้คือ:

- แตกต่างจากแพลตฟอร์มจองโรงแรมหรือจองทัวร์
- ใช้ AI ได้อย่างมีความหมาย
- ทำ MVP ได้จริง
- ต่อเป็นเกม ท่องเที่ยวชุมชน หรือระบบ B2B ได้
- เหมาะกับคนรุ่นใหม่ที่ชอบแชร์ประสบการณ์

แนวคิดหลักคือ:

```text
ไม่ได้บอกแค่ว่า “ไปไหน”
แต่ทำให้ผู้ใช้รู้สึกว่า “ทริปนี้เป็นของฉัน”
```
